1 Reconnaissance

Premier réflexe sur une cible Active Directory exposant SMB : lister les partages accessibles, y compris en session anonyme.

bash 
smbclient -L //cicada.htb/
liste des partages SMB
ℹ
Un partage HR attire l'attention, potentiellement accessible en session nulle, à tester en priorité.

2 Énumération

Partage HR — accès anonyme

Le partage HR est accessible sans authentification (session nulle) :

bash 
smbclient -N //cicada.htb/HR

Un fichier présent sur ce partage contient un mot de passe par défaut communiqué à l'ensemble des employés :

fichier du partage HR — mot de passe par défaut
⚠
Mot de passe récupéré : Cicada$M6Corpb*@Lp#nZp!8.Il reste à trouver un compte utilisateur valide pour le tester (probable réutilisation en tant que mot de passe par défaut du domaine).

Énumération du domaine avec les credentials trouvés

Le mot de passe par défaut est testé sur un compte identifié (michael.wrightson), ce qui permet une énumération authentifiée du domaine :

bash 
enum4linux-ng -A -u 'michael.wrightson' -p 'Cicada$M6Corpb*@Lp#nZp!8' 10.129.196.205
enum4linux-ng, énumération authentifiée

3 Foothold

L'énumération du domaine révèle un compte david.orelious, dont le mot de passe est trouvé dans la description du compte (pratique courante mais risquée en AD) :

⚠
Credentials : david.orelious / aRt$Lp#7t*VQ!3

Ce compte donne accès à un nouveau partage, DEV :

bash 
smbclient -U david.orelious //cicada.htb/DEV

Un fichier du partage DEV expose à son tour de nouveaux credentials, cette fois pour le compte emily.oscars :

partage DEV — nouvelles credentials
⚠
Credentials : emily.oscars / Q!3@Lp#M6b*7t*Vt

Ce compte permet d'obtenir un shell distant via WinRM :

bash 
evil-winrm -i cicada.htb -u emily.oscars -p 'Q!3@Lp#M6b*7t*Vt'
shell obtenu avec emily.oscars
🚩
user.txt récupéré en tant que emily.oscars

4 Élévation de privilèges

Une fois le shell obtenu, l'énumération des privilèges du compte emily.oscars révèle un privilège Windows particulièrement puissant : SeBackupPrivilege.

whoami /priv — SeBackupPrivilege activé
⚡
Vulnérabilité : SeBackupPrivilege permet de lire n'importe quel fichier du système en contournant les ACL NTFS y compris les ruches de registre SAM/SYSTEM ou la base NTDS.dit, ouvrant la voie à l'extraction des hashes du domaine. hackingarticles.in/windows-privilege-escalation-sebackupprivilege.

Ce privilège est exploité pour sauvegarder les ruches nécessaires à l'extraction des secrets locaux/domaine :

abus de SeBackupPrivilege

Les fichiers récupérés sont ensuite analysés avec pypykatz pour en extraire le hash NTLM du compte Administrator :

pypykatz : extraction du hash Administrator

Le hash récupéré permet une authentification directe par Pass-the-Hash, sans connaître le mot de passe en clair :

bash 
evil-winrm -i cicada.htb -u Administrator -H "2b87e7c93a3e8a0ea4a581937016f341"

# rooted
Administrator (Pass-the-Hash)
2b87e7c93a3e8a0ea4a581937016f341
Hash NTLM extrait via SeBackupPrivilege + pypykatz et accès Administrator obtenu

5 Ce que j'ai retenu

→
Toujours tester l'accès anonyme (session nulle) sur les partages SMB avant toute authentification : un mot de passe « par défaut » oublié dans un fichier RH peut suffire à démarrer toute la chaîne.
→
La réutilisation de credentials trouvés dans un partage pour accéder au suivant est un schéma très courant sur les box AD faciles : chaque nouvel accès doit être vu comme un tremplin vers d'autres partages ou comptes.
→
SeBackupPrivilege est un privilège à surveiller en priorité lors d'une énumération post-exploitation Windows : il permet de contourner les ACL NTFS et mène très vite à un dump complet des secrets du système.
→
pypykatz est une alternative pratique à Mimikatz pour extraire des secrets depuis des ruches de registre exfiltrées, sans dépendre d'un binaire Windows sur la cible.
