Déploiement silencieux d'un logiciel (MSI) via GPO
Distribuer automatiquement un logiciel (fichier .msi) sur un ensemble de postes du domaine, sans intervention utilisateur, via une stratégie de groupe dédiée.
Outils : GPMC, Group Policy Objects Durée estimée : 20–30 min Dernière mise à jour : juillet 2026
GPO Déploiement logiciel Active Directory

1 Contexte & objectif

Besoin de déployer un logiciel de manière homogène sur un groupe de postes (ex. un agent de sécurité) sans passer poste par poste, en s'appuyant sur l'infrastructure Active Directory déjà en place.

ℹ
Le déploiement via GPO nécessite un fichier .msi (l'installation "assignée" ne fonctionne pas avec un .exe sans wrapper).

2 Prérequis

  • Droits d'administration sur la GPMC (Group Policy Management Console).
  • Fichier .msi accessible depuis un partage réseau lisible par les ordinateurs cibles (ex. \\srv\partage$).
  • Une OU (unité d'organisation) regroupant les postes concernés.
⚠
Tester d'abord la GPO sur une OU de test avant de l'appliquer à la production.

3 Procédure

Étape 1 — Créer la GPO

Dans la GPMC, clic droit sur l'OU cible → Créer un objet GPO dans ce domaine, et le lier ici. Nommer la GPO clairement (ex. MSI_deploiement_logiciel).

Étape 2 — Configurer l'installation logicielle

Éditer la GPO → Configuration ordinateur → Stratégies → Paramètres du logiciel → Installation de logiciel. Clic droit → Nouveau → Package, puis sélectionner le fichier .msi via son chemin UNC réseau.

texte 
# Chemin réseau à utiliser dans l'assistant (UNC obligatoire, pas de lettre de lecteur)
\\srv-partage\logiciels$\monlogiciel\package.msi

Choisir la méthode Assigné pour une installation automatique au démarrage de la machine, sans action utilisateur.

assistant d'installation logicielle
[ Capture à insérer depuis Notion ]

Étape 3 — Lier et forcer l'application

Vérifier que la GPO est bien liée à la bonne OU, puis forcer une actualisation sur un poste de test.

powershell 
gpupdate /force
# Un redémarrage est nécessaire pour l'installation "assignée" côté ordinateur
Restart-Computer

4 Vérification

Après redémarrage du poste de test, contrôler que le logiciel est bien installé.

// résultat attendu
Le logiciel apparaît dans "Applications installées" et le journal d'événements (Applications and Services Logs → Group Policy) ne montre aucune erreur de déploiement.

5 Points d'attention & retour d'expérience

→
Toujours utiliser un chemin UNC (jamais une lettre de lecteur mappée) : le compte "SYSTEM" de l'ordinateur n'a pas accès aux lecteurs réseau de l'utilisateur.
→
Vérifier les droits NTFS/partage sur le dossier réseau : le groupe "Ordinateurs du domaine" doit avoir un accès en lecture.
→
Un déploiement "assigné" au niveau ordinateur nécessite un redémarrage ; au niveau utilisateur, une reconnexion suffit.
